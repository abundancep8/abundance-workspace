Routing Decision Tree:

  Task arrives from cron job
         ↓
  Is it analysis/batch/long-context?
  Or >10,000 tokens?
  Or non-time-sensitive?
         ↓YES
  Route to KIMI K2.5
  (Saves ~75% on cost)
         ↓NO
  Route to CLAUDE 3.5 SONNET
  (Maintains quality for critical work)

Cost Calculation:
  Kimi K2.5 via OpenRouter:
    Input:  $0.14 per 1M tokens
    Output: $0.42 per 1M tokens
  
  Claude 3.5 Sonnet via OpenRouter:
    Input:  $3.00 per 1M tokens
    Output: $15.00 per 1M tokens

  Formula: (input_tokens × input_rate) + (output_tokens × output_rate)

Performance Tracking:
  • Timestamp: When task executed
  • Model: Which model processed it
  • Input tokens: Tokens sent to model
  • Output tokens: Tokens generated by model
  • Execution time: Total time in milliseconds
  • Cost: Calculated based on token count and model rates
  • Success: Boolean (pass/fail)
