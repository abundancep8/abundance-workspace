# YouTube Comment Monitor Setup

## Prerequisites

1. **YouTube Data API v3 Credentials**
   - Go to [Google Cloud Console](https://console.cloud.google.com)
   - Create a new project or select existing
   - Enable YouTube Data API v3
   - Create API key (or OAuth 2.0 credentials for authenticated requests)

2. **Channel ID for Concessa Obvius**
   - Find at: https://www.youtube.com/@[channel]/about
   - Copy the channel ID from the URL or about page
   - Update `CONFIG.channelId` in `scripts/youtube-monitor.js`

## Installation

### 1. Set Environment Variable
```bash
export YOUTUBE_API_KEY="your-api-key-here"
```

To make it permanent, add to your shell profile (~/.zshrc or ~/.bashrc):
```bash
echo 'export YOUTUBE_API_KEY="your-api-key-here"' >> ~/.zshrc
```

### 2. Install Dependencies (if needed)
The script uses Node.js built-ins (https, fs). No npm packages required.

### 3. Create Cron Job

Edit your crontab:
```bash
crontab -e
```

Add this line to run every 30 minutes:
```cron
*/30 * * * * cd /Users/abundance/.openclaw/workspace && node scripts/youtube-monitor.js >> .cache/youtube-monitor.log 2>&1
```

Or for every 30 minutes starting at :00 and :30:
```cron
0,30 * * * * cd /Users/abundance/.openclaw/workspace && node scripts/youtube-monitor.js >> .cache/youtube-monitor.log 2>&1
```

Verify:
```bash
crontab -l
```

## Configuration

### Update Channel ID
In `scripts/youtube-monitor.js`, line 13:
```javascript
channelId: 'UC...' // Replace with Concessa Obvius channel ID
```

### Customize Template Responses
Edit `TEMPLATES` object (lines ~30-35) to match your voice:
```javascript
const TEMPLATES = {
  question: 'Your custom response to questions...',
  praise: 'Your custom response to praise...',
};
```

### Adjust Category Keywords
Modify `CATEGORIES` object (lines ~38-54) to match your audience:
- Add/remove keywords
- Adjust scoring logic

## Outputs

- **Log file:** `.cache/youtube-comments.jsonl` (one JSON object per line, all comments)
- **State file:** `.cache/youtube-monitor-state.json` (tracks last check time)
- **Cron log:** `.cache/youtube-monitor.log` (execution logs)

Check cron job status:

## API Limits

YouTube Data API v3:
- Default quota: 10,000 units/day
- commentThreads.list: ~3 units per request
- replies.insert (auto-respond): ~50 units per request
- At 30-min intervals: ~48 requests/day = ~144 units minimum

**This should fit within free tier**, but monitor usage in [Google Cloud Console](https://console.cloud.google.com/apis/dashboard).

## Troubleshooting

### "YOUTUBE_API_KEY not set"
```bash
export YOUTUBE_API_KEY="your-key"
node scripts/youtube-monitor.js
```

### "API error: 403 Forbidden"
- Check API key is valid
- Verify YouTube Data API v3 is enabled in Cloud Console
- Check quota not exceeded

### No comments found
- Verify channel ID is correct
- Check if channel has recent comments
- Verify API key has access to channel

### Cron job not running
```bash
# Check system log
log stream --predicate 'eventMessage contains[cd] "youtube-monitor"'

# Test cron manually
cd /Users/abundance/.openclaw/workspace && node scripts/youtube-monitor.js
```

## Auto-Responding (Production)

The current script logs responses but doesn't actually post them. To enable auto-responses:

1. Use OAuth 2.0 authentication (not API key)
2. Request proper YouTube scopes
3. Use `youtube.commentThreads.replies.insert` endpoint
4. Handle rate limiting and quota

This requires more setup but ensures responses come from your account, not an API key.
