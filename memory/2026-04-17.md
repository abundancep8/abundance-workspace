## Framework: Remove Friction Before Waiting for Human Action

**Lesson:** When a blocker depends on user action but the process is complex, build the guidance tool first. Don't wait passively for the action to happen — make it easier to do.

**Pattern:**
1. Identify blocker (YouTube OAuth setup)
2. Analyze friction points (5 manual steps, Google Cloud navigation, credential placement)
3. Build guidance tool (interactive script + reference guide)
4. Add auto-detection/migration (no follow-up steps needed)
5. Document clearly (expectations, timeline, troubleshooting)

This applies to:
- OAuth flows for APIs
- Configuration workflows
- Multi-step deployments
- Credential setup
- Any process that depends on user action but has high friction

**Result:** Reduces average time from "I should do this" to "It's done" from days to minutes.
