## Token Optimization Rules (LOCKED IN)

### 1. Model Selection Protocol
```
Default: anthropic/claude-haiku-4-5 (cheapest, fast)
Escalate to: anthropic/claude-sonnet-3.5 (complex reasoning only)
Never use: Opus unless absolutely necessary
Rule: Prove Haiku insufficient before escalating
```

### 2. Batch Operations
- Never make single API calls when batch possible
- Combine 5-10 similar requests → single LLM call
- Cache results for 24h reuse
- Reduce round-trips by 70%

### 3. Prompt Engineering
- Remove verbose instructions; use concise directives
- Pre-filter data before sending to LLM
- Use system prompts to reduce chat tokens
- Estimated savings: 20-30% per call

### 4. Caching Strategy
- Cache frequent recaps (Master Key teachings = reusable)
- Cache video intelligence frames
- Cache memory file structures
- Cache Obsidian formatting templates
- Don't re-analyze same content twice

### 5. API Call Minimization
- Web fetch over LLM analysis when possible
- Local processing before cloud operations
- Use regex/keyword matching instead of LLM categorization
- Video intelligence: download once, analyze multiple times

### 6. Budget Monitoring
- **Alert at 75% daily spend ($3.75)**
- **Hard stop at 100% ($5.00)**
- Announce spend every 2 hours
- Log all LLM calls with token count
- Weekly budget review Friday 8 PM

### 7. Research Agent Optimization
- Master Key playlist: analyzed once, never re-analyze
- Recaps stored in memory: reference, don't re-generate
- Obsidian notes: generated once, accessed via memory
- Result: Research agent token cost dropping to near-zero after initial processing

### 8. YouTube Monitoring Optimization
- Autonomous comment/DM monitor uses zero API tokens
- Local caching eliminates API calls
- Keyword categorization (regex-based)
- Auto-responses templated (no LLM generation per response)
- Estimated savings: 80% vs. API-based approach
